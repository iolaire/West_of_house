# Documentation Organization

This directory contains all project documentation organized by category.

## Directory Structure

### `/deployment/`
Deployment-related documentation including:
- AWS Amplify Gen 2 setup and configuration
- IAM permissions and security
- Resource tagging strategies
- Deployment verification and status reports
- Production deployment guides

**Key Files:**
- `AMPLIFY_GEN2_SETUP.md` - Complete Gen 2 deployment guide
- `IAM_CONFIGURATION.md` - IAM roles and permissions setup
- `AWS_RESOURCE_TAGGING.md` - Resource tagging best practices
- `DEPLOYMENT_VERIFICATION_COMPLETE.md` - Deployment verification checklist

### `/development/`
Development guides and best practices:
- Quick start guides
- Error handling improvements
- Domain safety considerations

**Key Files:**
- `QUICK_START.md` - Getting started with development
- `ERROR_HANDLING_IMPROVEMENTS.md` - Error handling patterns
- `DOMAIN_SAFETY.md` - Domain configuration safety

### Root Level (Game Design)
Game-specific design documentation:
- `game_overview.md` - High-level game design
- `HALLOWEEN_MECHANICS.md` - Halloween-themed game mechanics
- `HAUNTED_TRANSFORMATION.md` - Transformation from original Zork
- `FLAG_TRANSFORMATIONS.md` - Game flag system documentation

## Finding Documentation

**For Deployment:**
→ See `deployment/` folder

**For Development:**
→ See `development/` folder

**For Game Design:**
→ See root level files

**For Project Overview:**
→ See `/README.md` in project root

## Contributing

When adding new documentation:
1. Place deployment docs in `deployment/`
2. Place development guides in `development/`
3. Place game design docs in root level
4. Update this README if adding new categories
